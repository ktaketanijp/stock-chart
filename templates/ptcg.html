重要ルール・注意事項
提出ルール
  • 1日の最大提出回数: 5回
  • 同時アクティブな提出数: 最新2件
  • 提出形式: submission.tar.gz（main.py + deck.csv）
実装上の制約（必ず守ること）
  • エージェントはクラッシュしてはならない（常にfallback合法手を返す）
  • 1試合あたり持ち時間: プレイヤーごと10分以内（超過で自動敗北）
  • 合法手リスト外のインデックスを返してはならない
  • 外部API・LLM・ネットワークアクセス禁止
  • 使用カードは主催者指定のスタンダードリストのみ
エージェント関数の基本形（実装時の必須パターン）
def agent(obs_dict: dict) -> list[int]: try: # ここにロジック return [best_action_index] except Exception: # クラッシュ防止: 必ず合法手を返す return [0]
採点基準（Strategyレポート）
モデルアプローチの独創性
70%
ルールベース単体では高得点不可。MCTS・PPO・情報推論など何らかの学習・探索コンポーネントが必須。前向き計画（相手手札推定）があると高評価。
デッキ設計コンセプト
20%
なぜそのデッキを選んだか戦略的理由が必要。シンプルなデッキが実際に強い（bellibolt ELO 836実績）。
レポートの品質
10%
英語2000語以内。文章は翻訳で対応可。実験結果・数値・グラフを含めると高評価。
